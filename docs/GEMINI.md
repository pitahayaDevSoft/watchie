# Gemini CLI Rules

Specific instructions for the **Gemini CLI** agent.

## Working Context

- This project follows the FMG standard guidelines.
- Prioritize using `grep_search` to navigate the code.
- Always check `git status` before every commit.

## Workflow

1. Research -> 2. Strategy -> 3. Execution (Plan-Act-Validate). Don't skip steps.

## Restrictions

- Keep `CHANGELOG.md` updated with every `feat` or `fix`.
- Ensure all dependencies and features are defined in Cargo.toml.
